Deployment Strategy currently exposed publicly:
  Rolling Update - 3 replicas, maxSurge=1, maxUnavailable=0,
                   zero-downtime rolling updates, automatic
                   rollback via kubectl rollout undo.

All five required deployment strategies are deployed simultaneously
on the same Minikube cluster, each in a dedicated namespace:

  Strategy        Namespace          NodePort  Pods Running
  --------------- ------------------ --------- -----------------------
  Rolling Update  aceest-rolling     30081     3 stable
  Blue-Green      aceest-bluegreen   30082     2 blue + 2 green
  Canary          aceest-canary      30083     4 stable + 1 canary (80/20)
  A/B Testing     aceest-ab          30084     2 variant-A + 2 variant-B
  Shadow          aceest-shadow      30085     2 primary + 1 shadow

Due to the ngrok free-tier single-tunnel limitation, only the
rolling-update strategy has a public URL at submission time. All
five strategies' manifests are in k8s/ in the GitHub repo and are
documented in detail in the project report (docs/REPORT.docx).
